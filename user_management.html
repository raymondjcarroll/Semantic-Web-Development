Introduction to UNIX System Administration

User and Group Management

User Management

An important part of any OS is users. It's important to be able to restrict access to certain resources to certain people.

One of the unique features of UNIX, exist in most OSes is the idea of a superuser. This superuser is called root.

All accounts have a username, which is how you identify which user is who. All accounts must also have an UID. These UIDs must less than 65535.

Creating a User

FreeBSD has the command adduser to add users to the system. While it is possible to run this command with flags, adduser will run interactively by running it without any flags.

Note for adduser

adduser must be ran as superuser. To do this in FreeBSD, run sudo -i first.

Deleting a User

FreeBSD has the command rmuser to remove any user. You will be prompted to confirm the deletion.

Group Management

Creating a Group

FreeBSD also has the command pw that will allow you to create user groups. Use the flag groupadd to create a group.

An example syntax is sudo pw groupadd students to create a group called 'students'.

Adding a User to a Group

To add a user to a group, you can also use the pw command.

An example syntax is sudo pw group mod students -m raymond. This will add the user 'raymond' to the 'students' group.

Removing a User from a Group

The -d flag to the pw command allows deleting a user from a group.

An example syntax is sudo pw group mod students -d raymond to delete the user 'raymond' from the group 'students'.
